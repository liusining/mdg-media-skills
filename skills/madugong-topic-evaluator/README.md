# madugong-topic-evaluator

这是一个用于判断自媒体选题是否值得做的 Skill。

## 安装

在本仓库根目录运行：

```bash
npx --yes skills@latest add ./skills --all
```

这会从 `skills/` 安装本仓库提供的两个 Skill。

## 适用场景

用户给出一个选题描述，希望判断：

- 这个选题是否值得做。
- 它是否符合马督工选题方法论。
- 它的共同信息场、最新变化、行动建议分别是什么。
- 它命中八个选题方向中的哪些方向。
- 它是否会被“否定选题”规则淘汰。
- 它应该如何重构成更适合传播的选题。

## 核心流程

```text
用户给出选题描述
→ 抽取中心对象与暗含问题
→ 网络搜索背景、最新变化、矛盾/反证
→ 把选题重述为共同信息场 + 最新变化 + 行动建议
→ 等待用户确认
→ 按八个选题方向和否定选题规则评估
→ 输出结构化判断报告
```

## 设计模式

这个 Skill 组合了五类模式：

- **Pipeline**：保证搜索、确认、评估、输出顺序不可跳过。
- **Inversion**：先让用户确认三要素理解，再判断选题价值。
- **Tool Wrapper**：把马督工搜索方法、选题方法论放在 references 中按需加载。
- **Reviewer**：最终输出前用 checklist 自检。
- **Generator**：用固定模板输出理解确认和最终评估。

## 使用示例

用户：

```text
帮我判断这个选题：县城外卖骑手是不是正在成为农村中年男性的主要退路。
```

Skill 第一次输出：

```text
我先把这个选题理解成下面这个结构。请确认是否准确，确认后我再判断它是否值得做。

共同信息场：……
最新变化：……
行动建议：……

请确认：这个理解是否准确？
```

用户：

```text
确认，继续。
```

Skill 第二次输出最终评估：

```text
# 选题判断报告：……

## 1. 结论
**判断：值得做，但需要重构**
……
```

## 注意事项

- 这个 Skill 需要联网搜索工具。
- 用户未确认前，不应直接输出最终判断。
- 如果资料不足，应明确给出“暂缓”或“不建议做”，不要强行包装。
- 社交媒体只能用来观察舆论场，不能作为事实判断的唯一来源。
